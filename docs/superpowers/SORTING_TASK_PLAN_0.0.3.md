#prompt 
# 目的

目前服务已经完成话单分拣入库的大部分操作，本次需求需要完成后续操作。
- 前面开发中对于所有的文件操作和 io 操作均为模拟打桩，完成具体的代码实现逻辑，不用伪代码的方式，并通过测试用例验证。
- 前面话单已经从文件服务器读取并且入库，后续需要将话单数据推送给下游，推送方案未确定。
- 对于前面已经完成的话单分拣功能，需要补充几个功能:
	- 对于异常话单需要存储异常话单信息到异常文件表，并且记录异常原因。
	- 对于已经分拣过的话单包括异常话单（/backup /error目录下）需要定期清理，自动清理 创建时间在7 天以外话单文件。
	- 同时分拣话单移动到/backup 目录，/error 目录下时，需要细分根据日期创建目录，移动到对应目录下子日期目录下

# 约束

- 下游服务目的是接收所有话单包含的具体信息，目前使用量单位是按表的方式存储在本服务，后续考虑从代码中独立出去，归档到统一的配置服务。
- 话单数据推送给下游时，所有物理表的映射 ID 信息需要替换成对应的值，下游不接受对应 ID 重新映射
- 每次推送给下游数据控制数量，并且推送数据需要归档，已经推送的数据不能被重复推送。

#  验证

- 除本次新增功能点外，需要额外验证从话单采集分拣入库到推送下游的整个流程
- 检查流程中可用出现中断的场景，是否会出去流程中断后，无法重试或者恢复
